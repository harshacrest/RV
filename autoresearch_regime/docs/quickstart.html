Quick Start Guide

Regime Autoresearch + Meta-Harness — everything you need to run, configure, and monitor
🚀 3 Ways to Run
Option A
Inner Loop Only
The original autoresearch. A single Claude session modifies regime_experiment.py in a loop — edit, commit, evaluate, keep/revert. No meta-harness, no outer loop.
Claude prompt 
Read program_regime.md and kick off autoresearch. Tag: apr8
Option B
Meta-Harness (Automated)
MAB-adaptive outer loop. Automatically sweeps scoring weights, norms, features, and splits. Spawns a Claude subprocess for each inner loop. Evaluates on OOS.
Terminal 
python meta_harness.py --adaptive --budget 20 --max-inner 40
Option C
Meta-Harness (Claude-Driven)
Smarter than automated. Claude operates the outer loop — proposes configs, reads OOS results, reads execution traces, adapts strategy based on why things fail.
Claude prompt 
Read program_meta_harness.md and run the meta-harness
🎲 How the Two Loops Fit Together
META-HARNESS (Outer Loop) │ You / automated script controls the frame │ │ Writes meta_config.json → scoring weights, norms, features, splits, strategy weights │ │ ┌───────────────────────────────────────────────────────┐ │ │ AUTORESEARCH (Inner Loop) │ │ │ Claude modifies regime_experiment.py │ │ │ boundaries, features, weights → ~40 experiments │ │ │ evaluates on validation set (controlled by meta_config) │ │ └───────────────────────────────────────────────────────┘ │ │ After inner loop completes: │ best classifier → OOS evaluation → meta_results.tsv │ │ Upgrades: Knowledge Forest | MAB (5 arms) | IC Dedup | Co-Optimization | Paper Trade

The inner loop Claude never sees OOS data. It optimizes against the scoring formula in prepare_rv.py. The meta-harness changes that formula between iterations, then checks if the result generalizes.

What You Can Configure

⚙ Level 1: Scoring Formula & Frame meta_config.json

These control what the inner loop optimizes toward. The most impactful knobs in the system.

Scoring Weights (must sum to 1.0)
w_sharpe0.40Weight for portfolio Sharpe ratio. Higher = inner loop chases pure returns.
w_safe_sep0.25Weight for Safe vs Exposed gap. Higher = inner loop prioritizes regime discrimination.
w_rank_corr0.20Weight for train↔val stability. Higher = prevents overfitting to validation set.
w_coverage0.15Weight for state coverage. Penalizes if any state has <5 days in validation.
Normalization Constants
sharpe_norm5.0Sharpe divisor. Lower (3.0) = easier target, diversifies inner loop exploration. Try 3.0–7.0.
sharpe_cap1.5Max contribution from Sharpe term. Caps extreme Sharpe from dominating.
safe_sep_norm10.0Separation divisor. Lower (6.0) = easier target. Try 6.0–15.0.
safe_sep_cap1.0Max contribution from separation term.
Train / Validation Dates
train_start2023-02-01Training period start. Earlier = more data, but includes different regimes.
train_end2025-06-30Training period end.
val_start2025-07-01Validation period start. What the inner loop evaluates on.
val_end2026-01-30Validation period end.
Feature Pipeline
extra_features[]List of extra features to make available to the inner loop.
Available: IV_20d PK_20d IV_momentum_5d VRP_5d IV_range_10d RV_IV_gap PK_IV_zscore_60d IV_vol_of_vol_20d
Strategy Co-Optimization (Upgrade 4)
strategy_weights_override{}Dict of per-state [DM,WC,Orion] weights to inject before inner loop. Empty = no override.
strategy_lockFalseIf True, inner loop is told NOT to modify strategy weights — only optimize boundaries/features.
Templates: equal current_best wc_dominant orion_dominant regime_adaptive conservative
Inner Loop Budget
max_inner_experiments40Max experiments the inner loop Claude runs per iteration.
inner_timeout_sec3600Timeout in seconds (60 min).
inner_budget_usd5.0API cost cap per inner loop run.
# Create a custom config:
python -c "
from meta_config import MetaConfig
MetaConfig(
    w_sharpe=0.50, w_safe_sep=0.20, w_rank_corr=0.15, w_coverage=0.15,
    sharpe_norm=4.0,
    extra_features=['IV_momentum_5d', 'VRP_5d'],
    max_inner_experiments=30,
    description='sharpe-heavy-momentum'
).to_json('meta_configs/my_config.json')
"
🛠 Level 2: Upgrade Systems
Knowledge Forest
Persistent memory across all runs. Inner loop reads it before starting — avoids re-trying known failures.
experiment_forest.jsonAuto-created
python forest_manager.py --summary
python forest_manager.py --query "boundary"
MAB Scheduler
UCB1 bandit across 5 dimensions (scoring, norms, features, splits, strategy). Concentrates budget on what moves OOS Sharpe.
exploration_c1.41Higher = more exploration
python mab_scheduler.py --stats
IC Dedup Gate
Drops redundant features before inner loop. Pairwise Spearman > threshold gets filtered.
threshold0.75Lower = stricter dedup
python ic_dedup.py --features "IV_20d,PK_20d" --report
Paper Trade Gate
Forward-walk validation on most recent unseen data. Checks Sharpe, state diversity, transition stability.
days10Forward-walk days
min_sharpe-1.0Minimum Sharpe to pass
min_states2Minimum distinct states
python meta_harness.py --paper-trade-only --days 10
Strategy Co-Opt
Alternates between classifier and strategy optimization. The outer loop injects fixed weights; inner loop optimizes classifier only.
6 templates: equal, current_best, wc_dominant, orion_dominant, regime_adaptive, conservative
python meta_harness.py --sweep strategy --max-inner 40
python strategy_coopt.py --list-templates
python strategy_coopt.py --extract
📝 Level 3: Prompt Instructions

These markdown files are the AI agent's instructions. Edit them to change search strategy.

FileControlsWhat to Edit

🎯 Level 4: Classifier Internals regime_experiment.py

These are what the inner loop Claude modifies. You can also edit them directly before a run.

ParameterCurrent BestWhat It Does
IV_L1_UPPER8.5Below this = L1 (low IV). Lower = more L1 days.
IV_L2_UPPER11Below this = L2, above = L3. Controls L2/L3 split.
SPLIT_FEATURE_L3PK_IV_zscore_30dFeature that splits L3 into Safe/Exposed.
L2_DIRECTION_FEATUREIV_chg_1dIV direction signal at L2. Threshold: -1.1.
STRATEGY_WEIGHTS{L1 Safe: [0,1,0], ...}Per-state [DM, WC, Orion] allocation weights.
compute_extra_features()custom functionAdd any derived features. Full Python freedom.

Commands Reference

▶ Running
# ── Adaptive MAB sweep (recommended) ──
python meta_harness.py --adaptive --budget 20 --max-inner 40
python meta_harness.py --adaptive --budget 20 --paper-trade       # + forward validation
python meta_harness.py --adaptive --budget 5 --dry-run             # preview, no cost

# ── Single dimension sweep ──
python meta_harness.py --sweep scoring --max-inner 40    # 8 weight configs
python meta_harness.py --sweep norms --max-inner 40      # 6 norm configs
python meta_harness.py --sweep features --max-inner 40   # 7 feature sets
python meta_harness.py --sweep splits --max-inner 40     # 5 date splits
python meta_harness.py --sweep strategy --max-inner 40   # 6 strategy templates (co-opt)

# ── Single custom config ──
python meta_harness.py --config meta_configs/my_config.json
🔍 Diagnostics & Monitoring
# ── OOS baseline ──
python meta_harness.py --baseline

# ── Paper trade gate ──
python meta_harness.py --paper-trade-only --days 10

# ── Knowledge forest ──
python meta_harness.py --forest-summary
python forest_manager.py --query "boundary" --top 10
python forest_manager.py --failures
python forest_manager.py --stats

# ── MAB scheduler ──
python meta_harness.py --mab-stats

# ── Feature correlations ──
python ic_dedup.py --features "IV_20d,PK_20d,VRP_5d,RV_IV_gap" --report

# ── Strategy co-optimization ──
python strategy_coopt.py --list-templates                  # show all 6 templates
python strategy_coopt.py --extract                          # current weights in regime_experiment.py
python strategy_coopt.py --generate random                   # preview a random weight config

# ── Run classifier directly ──
python regime_experiment.py

🍳 Recipes
“Run overnight and let the system figure it out”
python meta_harness.py --adaptive --budget 30 --max-inner 40 --paper-trade
“Test if a different scoring formula generalizes better”
python meta_harness.py --sweep scoring --max-inner 40
# Check: sort meta_results.tsv by oos2_sharpe
“Test one specific scoring config”
python -c "
from meta_config import MetaConfig
MetaConfig(w_sharpe=0.30, w_safe_sep=0.30, w_rank_corr=0.25, w_coverage=0.15,
           description='equal-sharpe-sep').to_json('meta_configs/test.json')
"
python meta_harness.py --config meta_configs/test.json --max-inner 30
“Check if adding new features helps”
# First check correlations
python ic_dedup.py --features "IV_momentum_5d,VRP_5d,IV_range_10d" --report
# Then sweep
python meta_harness.py --sweep features --max-inner 40
“Find the best strategy allocation for a fixed classifier”
# Sweep all 6 strategy templates — inner loop only optimizes boundaries/features
python meta_harness.py --sweep strategy --max-inner 40

# Or test one specific template
python -c "
from meta_config import MetaConfig
from strategy_coopt import generate_strategy_config
MetaConfig(
    strategy_weights_override=generate_strategy_config('orion_dominant'),
    strategy_lock=True,
    description='coopt:orion'
).to_json('meta_configs/coopt_test.json')
"
python meta_harness.py --config meta_configs/coopt_test.json --max-inner 30
“Update the baseline after finding a better classifier”
cp regime_experiment.py regime_experiment_baseline.py
“Reset everything and start fresh”
rm -f meta_results.tsv experiment_forest.json mab_state.json meta_config.json
rm -rf meta_configs/iter_*.json
cp regime_experiment_baseline.py regime_experiment.py
