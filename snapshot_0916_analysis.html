2. How the 9:16 AM Snapshot Works

Snapshot Lag Logic

The regime system uses lagged features to predict today's returns (no look-ahead bias):

  • 3:30 PM (T-1): IV_lag = IV_1530.shift(1) → yesterday's close IV predicts today's return
  • 9:16 AM (T0): IV_lag = IV_0916.shift(-1).shift(1) → shift(-1) aligns to today's morning value, then shift(1) lags it → today's morning IV predicts today's return
Key difference: At 3:30 PM, you classify using information from ~18 hours ago. At 9:16 AM, you classify using information from right now — the morning open. This means 9:16 AM regimes can react to overnight gaps and morning IV spikes in real-time.
